# Laerdal Scenario Variable Analyzer

## Project Goal
Parse ~1 GB of Laerdal `.scx` simulator scenario files and produce a filterable React.js static web page showing statistics on which manikin control variables and custom variables are used across all scenarios.

The audience is a single internal user (Laerdal staff) wanting a personal overview of variable usage across the scenario library.

## Repository Layout
```
Scenario Data/
├── CLAUDE.md                  # this file
├── parser/                    # Node.js parser script
│   ├── parse.js               # entry point — reads .scx files, writes data.json
│   └── package.json
├── web/                       # Vite + React frontend
│   ├── src/
│   ├── public/
│   │   └── data.json          # generated by parser, read at runtime
│   └── package.json

```

## Tech Stack
- **Parser**: Node.js script (no framework, uses built-in `node:zlib` + `adm-zip` for ZIP, `fast-xml-parser` for XML)
- **Frontend**: Vite + React + TypeScript
- **Styling**: Tailwind CSS (responsive)
- **Data exchange**: `public/data.json` — parser writes it, React reads it at runtime

## .scx File Format
Each `.scx` file is a **ZIP archive** containing:

| File | Purpose |
|---|---|
| `ScenarioInfo.xml` | Manikin type, description, equipment list |
| `Scenario.xml` | States, events, manikin responses (variables) |
| `ScenarioPatient.xml` | Patient demographics |
| `SoundPresets.xml` | Audio config |
| `CtgPresets.xml` | CTG pattern config |
| `ScenarioRFIDMappings.xml` | Drug/RFID mappings |
| `Icons/`, `VocalSounds/` | Assets — ignore during parsing |

### Manikin Type
Found in `ScenarioInfo.xml`:
```xml
<AA:TargetedManikin>SimNewB2</AA:TargetedManikin>
<AA:AllowedManikins>
  <AA:Name>SimNewB</AA:Name>
</AA:AllowedManikins>
```
Use `TargetedManikin` as the primary manikin identifier.

### Variables in Scenario.xml
The XML namespace prefix is `AA` → `http://www.laerdal.com/Scenario`.

**Manikin control variables** (standard Laerdal responses):
```xml
<SingleValueResponse AA:Id="Laerdal.Response.Ecg.HeartRate_bpm" NewValue="47" ... />
<SingleValueResponse AA:Id="Laerdal.Response.SpO2.Value_Percent" NewValue="72" ... />
```

**Standard events** (instructor-triggered actions):
```xml
<Event AA:Id="Laerdal.Event.Start compressions" ... />
<BoolEvent AA:Id="Laerdal.Event.Attach pulse oximeter" ... />
<EnumEvent AA:Id="Laerdal.Event.Insert advanced airway" ... />
<DrugEvent AA:Id="Laerdal.Drug.EPINEPHRINE" ... />
```

**Custom variables/events** (scenario-specific, defined by scenario author):
```xml
<Event AA:Id="Custom.Event.Assess chest rise" ... />
<SingleValueResponse AA:Id="Custom.Response.SomeVariable" ... />
```

#### Variable Classification Rules
| Prefix | Type label |
|---|---|
| `Laerdal.Response.*` | Manikin Control Variable (standard) |
| `Laerdal.Event.*` | Standard Event |
| `Laerdal.Drug.*` | Drug Event |
| `Custom.Response.*` | Custom Manikin Variable |
| `Custom.Event.*` | Custom Event |

A variable's ID is unique enough to deduplicate across files (same `AA:Id` = same variable).

## data.json Schema
```jsonc
{
  "generated": "2026-05-01T12:00:00Z",
  "totalScenarios": 412,
  "variables": [
    {
      "id": "Laerdal.Response.Ecg.HeartRate_bpm",
      "category": "Laerdal.Response",   // top-level grouping
      "type": "standard",               // "standard" | "custom"
      "usedInCount": 312,               // number of .scx files that use it
      "usedInPercent": 75.7,            // percentage of totalScenarios
      "manikins": ["SimNewB2", "SimMan3G", "SimBaby"] // distinct manikins where used
    }
  ],
  "scenarios": [
    {
      "file": "000b6a0e...a40f22a6.scx",
      "manikin": "SimNewB2",
      "variableIds": ["Laerdal.Response.Ecg.HeartRate_bpm", "Custom.Event.End scenario"]
    }
  ]
}
```

## UI Filters
The React app must support filtering the variable table by:
- **Manikin** — multi-select, all manikins found in the dataset
- **Variable type** — Standard (Laerdal) / Custom
- **Variable category** — Response / Event / Drug
- **Min usage count** — slider or number input
- **Min % of scenarios** — slider or number input

## Common Commands
```bash
# Install parser dependencies
cd parser && npm install

# Run parser (regenerate data.json from .scx files)
node parser/parse.js --input "../Data" --output "../web/public/data.json"

# Install frontend dependencies
cd web && npm install

# Start dev server
cd web && npm run dev

# Build static site
cd web && npm run build
# Output in web/dist/ — open web/dist/index.html in a browser
```

## Notes
- `.scx` files may be nested in subfolders — the parser must walk directories recursively.
- The XML uses UTF-8 encoding; `SoundPresets.xml` may use UTF-16 — handle both.
- Module metadata is not currently available in the `.scx` files and is out of scope.
- The `Data/` folder (~1 GB) is not committed to the repo — it lives on the local machine.
- "Regenerate" workflow: re-run `parse.js`, then `npm run build` in `web/`.
